# Project Description

The alpha test for a strategic game app called Jacky In Darkadia has received several complaints from players. The app has several oversights in what aspects of the multiplayer game are handled in the server or on the client. This has led to several exploits using game mods to cheat by modifying values that should be immutable, such as health or damage, to make the game unplayably unbalanced for players not using game mods. There are also frequent performance issues, most notably frequent desyncs and connectivity problems. These oversights could also pose a huge cybersecurity risk to players who use the game's paid features.

We plan to address these challenges by building an app with a strong foundation in cybersecurity and a strategic delegation of tasks to the front and back end. The app will primarily focus on a multiplayer mode where a user can match with an opponent and play a real-time match against them. The app should include all of the functionality for the game's logic, as well as the framework for ongoing support and updates over time. The front-end is very important, as the game's design must be appealing and convey a strong sense of atmosphere without being taking on a computer's graphical resources. All of the logic that determines outcomes in the game should be handled by the server and communicated to both users without going out of sync, and be able to verify the game-state regularly to ensure all moves made are legal. 

The app will be hosted on Railway for the backend and Netlify for the frontend. Development will be carried out in Visual Studio Code. It uses Node.js and Express for the back-end and JavaScript for the front-end, with Flask API to communicate between Python and JS. Images will be deployed with Docker and rendered with HTML5.
